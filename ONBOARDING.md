## 1. Стек и репозиторий

- **Сайт:** `dreamhouse-site/` — Next.js 15.5.9 (App Router) + React 19 +
  TypeScript + Redux Toolkit + Tailwind v4. Шрифты Stetica / Inter /
  Manrope (`next/font/local`).
- **Это отдельный git-репозиторий** (не часть монорепо `DreamHouse2`).
  - Ветка: `main`
  - origin: `git@github-kelemetovrasid:kelemetovrasid/dreamhouse-new-sait.git`
  - Всё рабочее закоммичено и запушено в `origin/main`.
- **API:** внешний бэкенд `https://api.dreamhouse05.com/api`.
- **Источник истины по дизайну** детальной карточки/карты — мобильное
  Compose-приложение в монорепо (`features/.../feature_list_appartament`,
  токены `base/.../design/tokens/Dream*.kt`). CSS-переменные сайта
  (`app/globals.css`) 1:1 равны `DreamColors.kt`.

### Запуск
```bash
cd dreamhouse-site
npm run dev          # http://localhost:3000
```
> После правок, меняющих сигнатуры хуков, делай **жёсткую перезагрузку**
> страницы (Cmd+Shift+R) — иначе старый Fast-Refresh-чанк в памяти
> браузера даёт ложные баги (метки без носика, `useEffect changed size`).
> Если совсем странно — `rm -rf .next` + перезапуск dev.

## 2. Что сделано и работает

### Детальная карточка `/card/[id]` — основной маршрут
Pixel-perfect порт мобильного экрана DetailAppartament. Узкая колонка
по центру (max-width 480px), как телефон; десктоп = та же колонка.
Глобальный site-Header скрыт на `/card/` (у экрана свой UI).

Состав (порядок как в приложении):
- HeroGallery (4:5, скругление-низ, кнопки назад/шар/избр, счётчик)
- PriceTitleSection (розовая цена 24/700, ₽/м², заголовок, адрес,
  QuickInfoChips, бейдж «✓ Проверено»)
- DeveloperCardApp (GlassCard застройщика + подписка через Redux)
- UnderlineTabs: Описание · Характеристики · Документы · Карта
- Блок «Где находится»: встроенная **OpenStreetMap (MapLibre GL)** —
  Яндекс недоступен из РФ-сети, поэтому движок OSM. Кнопка «На карте»
  → Яндекс.Карты с маршрутом (`rtext=~lat,lng`), без координат →
  поиск по адресу. Если координат нет — плашка с адресом.
- «Возможно понравится»: карточки как на главной (`ListingsGrid`),
  fallback на объекты того же города если curations пуст; впритык к
  краям экрана.
- Sticky низ: Позвонить (tel:) + Отправить заявку (CallRequestModal).

Файлы: `app/card/[id]/page.tsx` + `app/card/[id]/lib/*`.
Все ссылки сайта (`/card/${id}`) уже ведут на этот дизайн.

### Карта `/map` — полноэкранная, MapLibre + OSM
Header/Footer/BottomBar скрыты (карта на весь экран `100dvh`, свой
MapTopBar). 11 продвинутых компонентов подключены:
MapCanvas, MapTopBar, SelectedCardSheet, NearbyCardsStrip,
MyLocationFAB, SearchThisAreaButton, ZoomBreadcrumb + `useMapCards`
хук + POI data.

Что работает:
- **Метки в едином синем стиле** `#0075FF` (одиночные и кластеры —
  одинаковая форма/размер/шрифт/носик, отличается только текст).
  Статус цветом: выбранная — розовая `#F1117E`, просмотренная —
  приглушённый тёмно-синий + opacity 0.6 (трекинг просмотров —
  localStorage `dh_viewed_cards`, пишется при открытии `/card/{id}`).
- **Кластеризация по координате**: квартиры одного ЖК (одна точка)
  схлопываются в маркер «от {минцена} ₽ · N кв.». Тап → шторка со
  **списком всех квартир ЖК**, тап по строке → карточка-превью.
- Превью-шторка (`SelectedCardSheet`): фото/цена/комнаты + кнопка
  адреса → Яндекс.Карты с маршрутом.
- Источник данных карты — `useMapCards` (НЕ Redux, единый источник).

Файлы: `app/map/page.tsx` + `app/map/components/*` + `app/map/hooks` +
`app/map/data`.

## 3. Процесс команды (как работаем)

Документирован в `TEAM_PROCESS_AND_PLAN.md`. Кратко на каждую задачу:

```
исследование → план(.md) → декомпозиция → реализация →
QA → РЕВЬЮ Tech-Lead'ом → коммит → отчёт
```

- **Tech-Lead (ревьюер)** — перед ним отчитываемся; правка не готова
  без его APPROVE. Ловит баги до прода (поймал, напр., удалённый
  `const fg` → `TS2304`, который проскочил мимо кэш-QA).
- **QA-чеклист (после каждой пачки):** `npx tsc --noEmit` без новых
  ошибок по затронутым файлам; `npx next lint` чисто; dev-лог
  `✓ Compiled` без ошибок; smoke `curl` затронутых роутов = 200;
  регрессия ссылок/соседних страниц; функционал сохранён.
- Один логический коммит = одна фича, осмысленное сообщение,
  `Co-Authored-By` в конце.

## 4. Известный техдолг и нюансы

- **Прод-сборка `npm run build` — ЗЕЛЁНАЯ ✅** (18/18 страниц
  пререндерятся). Ранее падала на 6 багах — все исправлены:
  AiBanner `variant`, leads.ts `AxiosError`, `useSearchParams`
  без `<Suspense>` в Header/layout/register/profile, и **отсутствие
  `maplibre-gl` в `package.json`** (использовалось из локального
  `node_modules` — клонирование репо на чистую машину падало с
  `Module not found`). Зафиксировано в `^5.24.0`.
  Остаётся 1 безобидный lint **warning** (не error, не блокирует
  build): `app/map/components/MapCanvas.tsx` poiMarkersRef cleanup
  в нетронутом POI-эффекте.
- **Перед каждым коммитом, меняющим зависимости/сборку, гонять
  чистый билд** (как у деплойщика):
  `rm -rf .next node_modules && npm install && npm run build`.
  Без этого `node_modules` с тёплым кэшом скрывает реальные баги
  (на эти грабли уже наступали).
- **Проверка сайта 2026-05-20**: 13 ключевых маршрутов → 200,
  все `✓ Compiled` без ошибок, 0 ссылок на удалённый `/card-v2`,
  tsc 0 / lint 0 errors, всё запушено в `origin/main` (`e548a2c`).
- **Яндекс.Карты недоступны** из части сетей (ERR_CONNECTION_CLOSED) —
  поэтому встроенные карты на OSM/MapLibre; внешние deep-link на
  Яндекс открываются в браузере пользователя (там обычно работает).
- ICard.latitude/longitude по типу `number`, но API может вернуть
  `null` — всегда runtime-guard перед использованием координат
  (см. комментарий в `app/types/models.ts`).
